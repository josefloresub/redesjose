Sección de preguntas sobre HTTP

  • 1) ¿Sobre qué protocolo binario está montado el protocolo HTTP?

    HTTP está montado sobre TCP (Transmission Control Protocol), que es un protocolo de transporte confiable en la capa de transporte del modelo OSI. HTTP también puede utilizar TLS (cuando se usa HTTPS), para encriptar la conexión sobre TCP.

  • 2) ¿Cuáles son los clientes HTTP y los servidores HTTP más utilizados?

    Clientes HTTP más utilizados: Navegadores web como Google Chrome, Mozilla Firefox, Microsoft Edge, y Safari. Herramientas de línea de comandos como cURL y Postman.

    Servidores HTTP más utilizados: Apache HTTP Server, Nginx, Microsoft IIS (Internet Information Services), LiteSpeed, Caddy.

  • 3) ¿Qué verbos admite un comando en el requerimiento HTTP?

    Los verbos HTTP, también conocidos como métodos, más comunes son: GET, POST, PUT, DELETE, HEAD, PATCH, OPTIONS.

  • 4) ¿Qué contenido lleva el body de un requerimiento HTTP?

    El cuerpo de una solicitud HTTP puede contener datos enviados al servidor, como: datos de formularios (por ejemplo, para solicitudes POST), JSON o XML (para intercambiar datos estructurados), archivos binarios (como imágenes o archivos de texto). Puede estar vacío en algunos casos, como en solicitudes GET o HEAD.

  • 5) ¿Qué diferencia existe entre un URL y un URI?

    URI (Uniform Resource Identifier) es un identificador genérico que puede ser un nombre, dirección o cualquier referencia a un recurso en la web. URL (Uniform Resource Locator) es un tipo de URI que especifica la ubicación de un recurso.

  • 6) ¿Cómo se almacena información relacionada con las respuestas HTTP?

    La información de las respuestas HTTP se puede almacenar usando: Cookies, headers de cache como Cache-Control y Expires, almacenamiento en caché en el servidor, o servidores proxy.

  • 7) ¿Qué significa Virtual Hosting?

    El Virtual Hosting permite que un solo servidor físico sirva múltiples dominios o sitios web. Existen dos tipos: Name-based Virtual Hosting y IP-based Virtual Hosting.

  • 8) ¿Por qué el HTTP se considera stateless?

    HTTP es considerado stateless porque cada solicitud es independiente y no mantiene información del estado de solicitudes anteriores.

  • 9) ¿Qué nueva versión de HTTP se está usando para mejorar la velocidad de la web?

    La nueva versión es HTTP/3, que usa QUIC en lugar de TCP, mejorando la velocidad y el rendimiento.

  • 10) ¿Cómo va a mejorar el comportamiento de una aplicación web en ambientes ruidosos o de débil conexión?

    HTTP/3 y QUIC mejoran la experiencia en conexiones inestables mediante la reducción de latencia y el manejo eficiente de paquetes perdidos.
